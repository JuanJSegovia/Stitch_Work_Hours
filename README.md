## Stitch Audit Log Analysis

###  Overview
This project analyzes a large Stitch audit trail to understand user activity, system usage patterns, and total time spent in the platform. The dataset includes more than 36,000 events generated by Stitch test accounts, internal users, and Salesforce system processes.

### What This Analysis Does
Cleans and filters raw audit log data
Keeps only relevant Stitch and test-user activity
Removes events tied to delegate access
Converts date/time fields into usable timestamps

###  Calculates:
Unique active days per user
Actions per day
Total hours active per day
Total hours across all accounts

###  Key Findings
A majority of activity comes from a primary Stitch test account.
Salesforce system accounts generate consistent automated events.
Combined user activity totals 900+ hours across the timeframe.
Several dates show very high action counts due to automation or bulk processes.

###  Visualizations
Two charts are included in the notebook:

###  Daily Action Counts
Total Active Hours Per Day
These help highlight usage spikes, trends, and patterns in development or testing activity.

###  Purpose
This analysis provides visibility into:
How frequently Stitch environments are used
How much activity is user-driven vs. automated
Daily workload and time‑in‑system metrics
Patterns that may support auditing, engineering review, or process optimization
